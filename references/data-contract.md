# 数据契约

> 描述转录文本的数据结构与分段格式约定,供 skill 处理字幕文件 / 转录结果时参考。
> 本契约移植自 BiliNote 项目 `backend/app/models/transcriber_model.py`,保持兼容,便于未来 L1/L2 阶段直接对接其转录脚本输出。

## TranscriptSegment(单段字幕)

| 字段 | 类型 | 说明 |
|---|---|---|
| `start` | float | 开始时间(秒) |
| `end` | float | 结束时间(秒) |
| `text` | str | 该段文字 |

## TranscriptResult(完整转录)

| 字段 | 类型 | 说明 |
|---|---|---|
| `language` | str? | 检测到的语言(如 `zh`、`en`) |
| `full_text` | str | 全部文字合并后的纯文本 |
| `segments` | Segment[] | 分段列表(带时间戳) |
| `raw` | dict? | 原始响应(调试用,可忽略) |

## segment_text 格式(喂给提示词的分段文本)

把 `segments` 转成每行一条的纯文本,格式**严格**为 `mm:ss - 文本`:

```
00:12 - 大家好,今天我们聊聊 RAG 的基本原理
00:45 - 首先要理解什么是检索增强生成
01:23 - 它的核心思路是把外部知识注入到提示词里
```

转换规则:

- 时间取 `start` 秒数,格式化为 `mm:ss`(两位数补零)。
- 超过 60 分钟用 `hh:mm:ss`。
- 每段一行,用 ` - `(空格-连字符-空格)分隔时间与文本。
- 该格式是 LLM 生成 `*Content-[mm:ss]` / `*Screenshot-[mm:ss]` 时间锚点的**唯一时间来源**,务必保留准确时间戳。

## 字幕文件解析参考

- `.srt`:序号行 → `HH:MM:SS,mmm --> HH:MM:SS,mmm` → 文本行(可能多行)→ 空行。取起始时间转秒,合并多行文本。
- `.vtt`:`WEBVTT` 头 → `HH:MM:SS.mmm --> HH:MM:SS.mmm` → 文本行。处理同 srt。
- `.txt`:无时间戳,按段落或句号切分,时间戳缺失时可省略(标记协议将降级为纯文本提示)。
